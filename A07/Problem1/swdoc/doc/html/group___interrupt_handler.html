void ScIsr ( void  )

Main interrupt handler

The interrupt handler is specific to the processor core and the compiler, as it depends on the stack layout

Depending on the interrupt controller implementation we can have multiple handlers for different vector or a single handler with chained functions. The latter is the case for HC08: when you add new interrupt services, they must be added to this handler.

Interrupt vector number specifies hardware address in vector table. For HC08, the external HW interrupt is number 2

This is the only scheduler function where user supplied code has to be added

Every serviced interrupt has to clear its source via the appropriate action (e.g. write to the clear register in case of n timer interrupt) and to call ScRescheduleFromIsr() as the last step of its processing. This will basically terminate the interrupt handler and jump to the scheduler. If there are still unserviced interrupts the interrupt handler will be called again.
