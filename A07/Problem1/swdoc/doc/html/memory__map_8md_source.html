5 ## Program memory
6 Program memory is separated into 2 distinct areas. The upper range, up to 0xffff is used a ROM and will be initialized when the VHDL code is compiled. The ROM can be updated using a special utility which downloads the binary directly into the assosciated block RAMs. The area below is the data area, which gets initialized by the program startup code. It keeps all data and variables, with sme exceptions as described under [tracing](@ref Tracing).
7 
8 The size of both areas can be specified via VHDL generics in multiples of 4kB (powers of 2 * 4K), the default is 8 blocks (32 kB) instruction and 2 blocks (8 kB) data.
9 The same settings must be used by the compile script.
10 
11 ### VHDL Specification
12 The generics `ni` and `nd` specify the number of 4k blocks for instruction (ni) and data (nd) ram.
13 
14  entity hc08_mini_cyc is
15  generic (
16  ni: integer := 8; -- number of 4k blocks instruction ram, must be power of 2
17  nd: integer := 2; -- number of 4k blocks data ram
18  ... more generics
19  simulation: boolean := false
20  );

23 ### Compiler Settings
24 To match the compiler settings with the VHDL configuration, specify `CSIZE` and `DSIZE` as follows in the build script:
25 
26  #### cyc1000: 32k rom, 8k ram
27  export CSIZE=0x8000
28  export DSIZE=0x2000
29  export CLOC=$((0x10000 - $CSIZE))
30  export SLOC=$(($CLOC - 1))
31  export DLOC=$(($CLOC - $DSIZE))
32  # we need to pass the DSEG location for the framebuffer
33  export CFLAGS="--stack-auto --stack-loc $SLOC -I $INC -DXSEG=$DLOC "
34  export LFLAGS="-b HOME=$CLOC -C $CSIZE -b XSEG=$DLOC -X $DSIZE -b DSEG=0x0000 -I 0x80"

39 ## Peripherals
40 All basic peripherals are mapped into the address range from 0x80 to 0xff. The range below 0x80 is resevered for RAM and keeps for example the kernel related variables like task, mailbox and mutex states.
41 
42 Therefore, the peripherals can be acessed using the fast HC08 direct addressing mode, which only works in page 0 (0x00 .. 0xff).
43 
44 The external SDRAM can be accessed via a register set which allows to read or write sequentially, after setting the start address.
